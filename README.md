# 주식 뉴스 텔레그램 봇

관심 종목의 뉴스와 시세를 텔레그램으로 받아보는 봇입니다. **Cloudflare Workers(무료)에서 실행**되며, 뉴스(구글 뉴스 RSS)와 시세(네이버 금융 공개 데이터) 모두 **API 키가 필요 없습니다**.

> **운영 구조 변경 (2026-06-12):** 처음에는 GitHub Actions로 운영했으나 무료 계정의 크론 스케줄이 2~5시간씩 지연되는 문제가 있어 Cloudflare Workers로 이전했습니다. 현재 실 운영 코드는 [worker/](worker/) 디렉터리이며, 파이썬 코드는 참고용으로 남겨두었습니다. 명령은 텔레그램 웹훅으로 **즉시** 처리되고, 브리핑·속보는 크론 트리거로 정시에 실행됩니다.

- **매일 설정 시간(기본 7시 KST)**: 종목별 최근 24시간 주요 뉴스 브리핑
- **설정 주기(기본 10분)마다**: 속보 키워드 기사 즉시 알림
- **텔레그램 명령어**로 종목·시간·주기 변경과 현재가 조회 가능

## 텔레그램 명령어

봇 채팅창에 입력하면 됩니다. 웹훅 방식이라 **응답이 즉시** 옵니다.

| 명령 | 설명 |
|---|---|
| `/list` | 현재 설정 보기 |
| `/price` | 등록 종목 전체 현재가 조회 |
| `/price 카카오` | 특정 종목 현재가 (미등록 종목도 가능) |
| `/add 카카오` | 종목 추가 |
| `/add 카카오 카카오 OR 카카오페이` | 뉴스 검색어를 직접 지정해서 추가 |
| `/del 카카오` | 종목 삭제 |
| `/time 8` | 브리핑 시간 변경 (24시 기준 → 매일 8시) |
| `/period 30` | 속보 체크 주기 변경 (분 단위, 최소 10분) |
| `/pause` | 브리핑·속보 일시정지 |
| `/resume` | 재개 |
| `/kw` | 속보 키워드 보기 |
| `/kw_add 상한가` / `/kw_del 상한가` | 속보 키워드 추가/삭제 |
| `/help` | 도움말 |

## 구조

```
.github/workflows/bot.yml   10분마다 실행되는 GitHub Actions 워크플로
bot.py                      메인 진입점: 명령 처리 → 브리핑 → 속보 체크
commands.py                 텔레그램 명령 처리
briefing.py                 뉴스 브리핑 생성·전송
breaking.py                 속보 감지·전송 (중복 방지 포함)
prices.py                   네이버 금융 현재가 조회
news.py                     구글 뉴스 RSS 수집
notify.py                   텔레그램 전송
settings.json               사용자 설정 (명령으로 변경되면 자동 커밋됨)
state/                      실행 상태 (보낸 기사 기록 등, 자동 커밋됨)
```

매 실행마다: ① 쌓인 텔레그램 명령 처리 → ② 브리핑 시간이 지났고 오늘 안 보냈으면 브리핑 전송 → ③ 속보 체크 주기가 됐으면 새 기사 확인. 변경된 설정·상태는 저장소에 자동 커밋됩니다.

## 처음 설정하는 방법 (포크해서 쓰는 경우)

1. **텔레그램 봇 생성**: @BotFather에서 `/newbot` → 토큰 발급 → 봇에게 메시지 한 번 전송 → `https://api.telegram.org/bot<토큰>/getUpdates`에서 chat_id 확인
2. **저장소 Secrets 등록**: Settings → Secrets and variables → Actions에 `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` 추가
3. **테스트**: Actions 탭 → `bot` 워크플로 → Run workflow (force_briefing 체크 시 브리핑 즉시 전송)

## 로컬 테스트

```
pip install -r requirements.txt
python briefing.py    # 토큰 없이 실행하면 전송 대신 콘솔에 출력됨
python breaking.py
```

## 알아둘 점

- GitHub Actions 무료 스케줄러는 혼잡 시 몇 분~십수 분 지연될 수 있습니다. 속보·명령 응답이 보통 10~20분 내라고 보면 됩니다.
- `/period`는 스케줄러 최소 주기 때문에 10분 미만으로 설정할 수 없습니다.
- 더 빠른 실시간성이 필요하면 Oracle Cloud Free Tier(무료 VM, 카드 등록 필요)로 옮겨 폴링 주기를 1분으로 줄일 수 있습니다.
